① 项目概述
游戏类型：2D 上帝俯视角（Top-Down）末世混沌割草生存游戏
核心 slogan：“在混沌房间中抉择，用秩序终结末世”
痛点对标：传统肉鸽重复地图、技能联动单一、策略反馈不清晰
开发周期：1 个月（MVP 可试玩）
团队配置：Solo 开发（你作为 U3D 主程：核心玩法+架构；美术/音效使用免费资源与轻量自制）
关键机制（优化后）（必须完整包含）

1. 与传统肉鸽的差异化创新点
独立房间策略：每一步都要在“普通/特殊/遗物”中做抉择，且强行把重复地图成本从体验里移除。
秩序/混沌双值系统：秩序是可控策略的货币，混沌是失败时“永久升级的惩罚”，形成张力。
技能-房间联动机制：技能不是纯数值升级，而是触发条件依赖房间属性；匹配才进化，不匹配反而会把负面拖到下一局面。
2. 关键交互细节（MVP 也能做）
长按调整技能释放范围：长按 0.2s~1.2s 动态扩大“精神立场/环形伤害范围”，松手释放；同时根据是否匹配联动房决定：
匹配时：范围提升带来进化收益
不匹配时：范围提升也会放大“负面视野限制”的体感
拖拽道具组合新装备（可先用“拖拽合成 UI”做轻量版）：把两件道具拖拽到“组合槽”，生成装备（装备会改变武器伤害或技能底层效果参数）。
房间选择 UI：结束房间后展示“两个候选房间类型卡”，显示消耗/收益与秩序值预览提示。
3. 难度曲线（用数据表达，方便调参）
第 1-2 次房间：引导区（70%胜率目标），确保玩家理解“秩序值预览”和“技能匹配触发”
第 3-5 次房间：挑战区（50%胜率目标），特殊联动房开始出现“绝对压制”类负面挑战
第 6-8 次房间：深度区（30%胜率目标），混沌负面会永久附着并导致战斗节奏变慢/更难
第 5 次房间：解锁隐藏机制（例如：遗物选择房出现“能降低基类混沌衰减阈值”的遗物）


关键机制

1. 独立房间系统（替代无限地图）
房间类型
普通战斗房：清群怪 → 获“技能等级/血量/武器伤害”升级选项（3 选 1）
特殊联动房：仍具有普通房间的战斗性质，带专属属性（如“绝对压制”），需匹配技能才能触发正向联动
遗物选择房：三选一遗物（改变全局策略，如“秩序值获取加速”）
房间选择策略
每个房间结束后可选择下次进入的房间
可消耗秩序值预览两个房间中的一个房间类型（普通/特殊/遗物）
选择“进”（挑战）或“退”（重选）
2. 秩序/混沌双值系统（核心驱动）
秩序值（Order）
获取：房间胜利（尤其联动房胜利）、技能进化
用途：
预览下一房间的两个候选中的一个（策略性选择）
规避负面联动效果（如“绝对压制”房的视野限制，通过秩序机制减轻体感）
混沌值（Chaos）
获取：房间失败时选“奋力一搏”
效果：负面效果永久附着
加强规则：再次遇同类房间时负面加强（例如：行动限制时间 +50%）
清除规则：
房间胜利时，当前所有负面效果清除
后续造成的负面效果会减弱（根据“基类混沌值”大小计算减弱）
并获相应提升（如技能伤害加成，或质的改变）
3. 技能-房间联动机制（差异化爽点）
底层逻辑
技能选择“底层效果”（如“精神立场”=环形伤害范围）
仅在匹配房间（如“绝对压制”房）触发联动
联动结果
不匹配：
仅触发负面效果
例如“精神立场”在不匹配房 → 视野受限（体感负反馈）
匹配且胜利：
“精神立场”进化（增加伤害 + 秩序值产出等）
获额外奖励
匹配且失败：
选择“奋力一搏” → 混沌值叠加
负面效果永久附着（并在下次同类房间更强）
4. 失败选项（风险与收益）
放弃：直接结束游戏，保留当前进度
奋力一搏：
收益：有机会逆转局势（例如“精神立场”进化）
风险：增混沌值 → 负面效果永久附着
即使切换房间也存在；再次失败则效果持续加强


③ 系统框架（U3D技术实现）
核心实现路径（Unity 组件/脚本拆解）
玩法驱动层（Game Flow）：房间生成、房间选择、胜负结算、秩序/混沌更新、联动触发
战斗层（Battle）：角色移动、敌人生成与AI、伤害计算、技能释放与范围/冷却
策略层（Strategy）：技能升级 3 选 1、遗物 3 选 1、道具拖拽合成、房间预览与进退决策
数据层（Data）：ScriptableObject 定义静态数据、运行时状态序列化到 JSON/PlayerPrefs
2D 建议：用 Rigidbody2D/Collider2D 做移动与命中；敌人用简化寻路或朝向跟随（MVP 不必上完整 NavMesh）。

生成技能联动规则表与解析器

生成 UI 视图模型（ViewModel）与 UI 状态机
尤其房间结算界面：展示秩序/混沌变化、联动成功/失败反馈
数据存储（MVP 简化可上线）
PlayerPrefs：保存小数据（当前关卡/房间数/解锁开关/最近存档索引）
JSON：保存完整状态（Order/Chaos、混沌附着负面列表、技能与遗物列表、当前房间类型等）
Unity Cloud Save：可在后期接入多端同步（MVP 先用本地 JSON）
性能优化要点（割草类必须提前管）
对象池：子弹/投射物/敌人（尽量复用而不是 Instantiate/Destroy）
Sprite 合批：使用 SpriteAtlas，减少材质切换（降低 DrawCall）
技能特效：统一用 ParticleSystem 池或轻量自绘（避免频繁生成）
敌人更新频率：远处敌人降低 AI tick（例如 0.1s 更新一次）
UI 频率：房间选择与结算 UI 只在状态变化时刷新
④ 美术与音效风格
美术方向：手绘精致微像素混搭卡通扁平化（轮廓细描边 + 简洁贴图），适配同屏 100+ 怪
色彩方案：
基底：#1A1A1A（墨黑）、#2D0A3A（深暗紫）、#3B0C0C（腐暗红示例）
高亮：#00FFFF（霓虹青）、#FF6A00（烈焰橙）、#7A2CFF（邪能紫）
音效需求：免费资源强化辨识度
连锁跳转、技能触发、房间切换
混沌连锁音效可用类似 electric_zap_01 的素材做变调/音高偏移
BGM：1 段简易循环（暗黑电子 + 低沉鼓点），压抑→爽感节奏可通过混音层叠实现（后期再上动态混音）
⑤ 测试与迭代计划
测试类型
单元测试（Test Framework）：
技能伤害计算
秩序/混沌更新与清除规则
技能-房间联动匹配判断
混沌附着负面加强逻辑（同类房间多次失败）
Playtest（用户测试）：招募 10-20 名目标用户
关注“玩家是否理解抉择成本”“失败后是否觉得可逆但有代价”
迭代节点
每 2 周版本评审，重点优化“核心循环的爽感”
例如调整“房间胜利反馈”“进化提示”“秩序值获取节奏”
BUG 优先级
致命：崩溃必须 24 小时内修复
严重：逻辑错导致核心玩法失效
次要：表现类 Bug，允许延后
⑥ 风险评估与应对
风险：AI 生成内容重复（对话/小任务同质化）
应对：Cursor 用“模板+随机种子+人工审核”生成变体，MVP 只需少量关键短句
风险：Unity 版本兼容问题
应对：锁定 Unity LTS 版本；CI/构建脚本固定
风险：秩序/混沌系统公式导致数值崩坏（要么太容易清负面、要么太难逆转）
应对：把公式抽成纯函数并做单元测试；用 Playtest 快速校准阈值
